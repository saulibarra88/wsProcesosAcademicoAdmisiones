# Reglas del Asistente

## Proceso de Planificación y Aprobación
- **Plan Obligatorio**: Siempre debes presentar un plan de implementación detallado (`implementation_plan.md`) al usuario antes de realizar cualquier cambio en el código, instalar dependencias o ejecutar comandos que modifiquen el entorno.
- **Esperar Aprobación**: No realices modificaciones de archivos ni ejecutes comandos de ejecución (como instalaciones, builds, scripts de prueba modificadores) hasta que el usuario haya revisado y aprobado explícitamente el plan de implementación.
